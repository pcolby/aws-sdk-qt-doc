69 /**
70  * @class SfnClient
71  *
72  * @brief Client for AWS Step Functions ( SFN)
73  *
74  * <fullname>AWS Step Functions</fullname>
75  *
76  * AWS Step Functions is a service that lets you coordinate the components of distributed applications and microservices
77  * using visual
78  *
79  * workflows>
80  *
81  * You can use Step Functions to build applications from individual components, each of which performs a discrete function,
82  * or <i>task</i>, allowing you to scale and change applications quickly. Step Functions provides a console that helps
83  * visualize the components of your application as a series of steps. Step Functions automatically triggers and tracks each
84  * step, and retries steps when there are errors, so your application executes predictably and in the right order every
85  * time. Step Functions logs the state of each step, so you can quickly diagnose and debug any
86  *
87  * issues>
88  *
89  * Step Functions manages operations and underlying infrastructure to ensure your application is available at any scale.
90  * You can run tasks on AWS, your own servers, or any system that has access to AWS. You can access and use Step Functions
91  * using the console, the AWS SDKs, or an HTTP API. For more information about Step Functions, see the <i> <a
92  * href="http://docs.aws.amazon.com/step-functions/latest/dg/welcome.html">AWS Step Functions Developer Guide</a>
93  */
