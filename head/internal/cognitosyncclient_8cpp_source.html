69  *
70  * <fullname>Amazon Cognito Sync</fullname>
71  *
72  * Amazon Cognito Sync provides an AWS service and client library that enable cross-device syncing of application-related
73  * user data. High-level client libraries are available for both iOS and Android. You can use these libraries to persist
74  * data locally so that it's available even if the device is offline. Developer credentials don't need to be stored on the
75  * mobile device to access the service. You can use Amazon Cognito to obtain a normalized user ID and credentials. User
76  * data is persisted in a dataset that can store up to 1 MB of key-value pairs, and you can have up to 20 datasets per user
77  *
78  * identity>
79  *
80  * With Amazon Cognito Sync, the data stored for each identity is accessible only to credentials assigned to that identity.
81  * In order to use the Cognito Sync service, you need to make API calls using credentials retrieved with <a
82  * href="http://docs.aws.amazon.com/cognitoidentity/latest/APIReference/Welcome.html">Amazon Cognito Identity
83  *
84  * service</a>>
85  *
86  * If you want to use Cognito Sync in an Android or iOS application, you will probably want to make API calls via the AWS
87  * Mobile SDK. To learn more, see the <a
88  * href="http://docs.aws.amazon.com/mobile/sdkforandroid/developerguide/cognito-sync.html">Developer Guide for Android</a>
