Client for AWS OpsWorks.

<fullname>AWS OpsWorks</fullname>

Welcome to the AWS OpsWorks Stacks API Reference. This guide provides descriptions, syntax, and usage examples for AWS OpsWorks Stacks actions and data types, including common parameters and error codes.

</p

AWS OpsWorks Stacks is an application management service that provides an integrated experience for overseeing the complete application lifecycle. For information about this product, go to the AWS OpsWorks details page.

</p

SDKs and CLI

</p

The most common way to use the AWS OpsWorks Stacks API is by using the AWS Command Line Interface (CLI) or by using one of the AWS SDKs to implement applications in your preferred language. For more information,

see>

  • AWS CLI

    </p

  • AWS SDK for Java

    </p

  • AWS SDK for .NET

    </p

  • AWS SDK for PHP 2

    </p

  • AWS SDK for Ruby

    </p

  • AWS SDK for Node.js

    </p

  • AWS SDK for Python(Boto)

    </p

Endpoints

</p

AWS OpsWorks Stacks supports the following endpoints, all HTTPS. You must connect to one of the following endpoints. Stacks can only be accessed or managed within the endpoint in which they are

created>

  • opsworks.us-east-1.amazonaws.co>

  • opsworks.us-east-2.amazonaws.co>

  • opsworks.us-west-1.amazonaws.co>

  • opsworks.us-west-2.amazonaws.co>

  • opsworks.ca-central-1.amazonaws.com (API only; not available in the AWS

    console>

  • opsworks.eu-west-1.amazonaws.co>

  • opsworks.eu-west-2.amazonaws.co>

  • opsworks.eu-west-3.amazonaws.co>

  • opsworks.eu-central-1.amazonaws.co>

  • opsworks.ap-northeast-1.amazonaws.co>

  • opsworks.ap-northeast-2.amazonaws.co>

  • opsworks.ap-south-1.amazonaws.co>

  • opsworks.ap-southeast-1.amazonaws.co>

  • opsworks.ap-southeast-2.amazonaws.co>

  • opsworks.sa-east-1.amazonaws.co>
