69 /**
70  * @class DataPipelineClient
71  *
72  * @brief Client for AWS Data Pipeline
73  *
74  * AWS Data Pipeline configures and manages a data-driven workflow called a pipeline. AWS Data Pipeline handles the details
75  * of scheduling and ensuring that data dependencies are met so that your application can focus on processing the
76  *
77  * data>
78  *
79  * AWS Data Pipeline provides a JAR implementation of a task runner called AWS Data Pipeline Task Runner. AWS Data Pipeline
80  * Task Runner provides logic for common data management scenarios, such as performing database queries and running data
81  * analysis using Amazon Elastic MapReduce (Amazon EMR). You can use AWS Data Pipeline Task Runner as your task runner, or
82  * you can write your own task runner to provide custom data
83  *
84  * management>
85  *
86  * AWS Data Pipeline implements two main sets of functionality. Use the first set to create a pipeline and define data
87  * sources, schedules, dependencies, and the transforms to be performed on the data. Use the second set in your task runner
88  * application to receive the next task ready for processing. The logic for performing the task, such as querying the data,
89  * running data analysis, or converting the data from one format to another, is contained within the task runner. The task
90  * runner performs the task assigned to it by the web service, reporting progress to the web service as it does so. When
91  * the task is done, the task runner reports the final success or failure of the task to the web
92  */

331  *
