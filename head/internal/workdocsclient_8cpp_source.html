111 /**
112  * @class WorkDocsClient
113  *
114  * @brief Client for Amazon WorkDocs
115  *
116  * The WorkDocs API is designed for the following use
117  *
118  * cases> <ul> <li>
119  *
120  * File Migration: File migration applications are supported for users who want to migrate their files from an on-premises
121  * or off-premises file system or service. Users can insert files into a user directory structure, as well as allow for
122  * basic metadata changes, such as modifications to the permissions of
123  *
124  * files> </li> <li>
125  *
126  * Security: Support security applications are supported for users who have additional security needs, such as antivirus or
127  * data loss prevention. The API actions, along with AWS CloudTrail, allow these applications to detect when changes occur
128  * in Amazon WorkDocs. Then, the application can take the necessary actions and replace the target file. If the target file
129  * violates the policy, the application can also choose to email the
130  *
131  * user> </li> <li>
132  *
133  * eDiscovery/Analytics: General administrative applications are supported, such as eDiscovery and analytics. These
134  * applications can choose to mimic or record the actions in an Amazon WorkDocs site, along with AWS CloudTrail, to
135  * replicate data for eDiscovery, backup, or analytical
136  *
137  * applications> </li> </ul>
138  *
139  * All Amazon WorkDocs API actions are Amazon authenticated and certificate-signed. They not only require the use of the
140  * AWS SDK, but also allow for the exclusive use of IAM users and roles to help facilitate access, trust, and permission
141  * policies. By creating a role and allowing an IAM user to access the Amazon WorkDocs site, the IAM user gains full
142  * administrative visibility into the entire Amazon WorkDocs site (or as set in the IAM policy). This includes, but is not
143  * limited to, the ability to modify file permissions and upload any file to any user. This allows developers to perform
144  * the three use cases above, as well as give users the ability to grant access on a selective basis using the IAM
145  */
