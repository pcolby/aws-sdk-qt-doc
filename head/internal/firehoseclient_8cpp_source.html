115  * Creates a delivery

117  * stream>

119  * By default, you can create up to 20 delivery streams per

121  * region>
122  *
123  * This is an asynchronous operation that immediately returns. The initial status of the delivery stream is
124  * <code>CREATING</code>. After the delivery stream is created, its status is <code>ACTIVE</code> and it now accepts data.
125  * Attempts to send data to a delivery stream that is not in the <code>ACTIVE</code> state cause an exception. To check the
126  * state of a delivery stream, use
127  *
128  * <a>DescribeDeliveryStream</a>>
129  *
130  * A Kinesis Firehose delivery stream can be configured to receive records directly from providers using <a>PutRecord</a>

132  * stream as input, set the <code>DeliveryStreamType</code> parameter to <code>KinesisStreamAsSource</code>, and provide
133  * the Kinesis stream ARN and role ARN in the <code>KinesisStreamSourceConfiguration</code>
134  *
135  * parameter>
136  *
137  * A delivery stream is configured with a single destination: Amazon S3, Amazon ES, or Amazon Redshift. You must specify
138  * only one of the following destination configuration parameters: <b>ExtendedS3DestinationConfiguration</b>,

140  *
141  * <b>RedshiftDestinationConfiguration</b>>
142  *
143  * When you specify <b>S3DestinationConfiguration</b>, you can also provide the following optional values:
144  * <b>BufferingHints</b>, <b>EncryptionConfiguration</b>, and <b>CompressionFormat</b>. By default, if no
145  * <b>BufferingHints</b> value is provided, Kinesis Firehose buffers data up to 5 MB or for 5 minutes, whichever condition
146  * is satisfied first. Note that <b>BufferingHints</b> is a hint, so there are some cases where the service cannot adhere
147  * to these conditions strictly; for example, record boundaries are such that the size is a little over or under the
148  * configured buffering size. By default, no encryption is performed. We strongly recommend that you enable encryption to
149  * ensure secure data storage in Amazon
150  *
151  * S3>

160  *
161  * parameter> </li> <li>
162  *
163  * The compression formats <code>SNAPPY</code> or <code>ZIP</code> cannot be specified in

166  *
167  * formats> </li> <li>
168  *
169  * We strongly recommend that you use the user name and password you provide exclusively with Kinesis Firehose, and that
170  * the permissions for the account are restricted for Amazon Redshift <code>INSERT</code>
171  *
172  * permissions> </li> </ul>
