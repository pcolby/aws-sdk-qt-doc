61 /**
62  * @class OpsWorksCMClient
63  *
64  * @brief Client for AWS OpsWorks for Chef Automate (OpsWorksCM)
65  *
66  * <fullname>AWS OpsWorks CM</fullname>
67  *
68  * AWS OpsWorks for configuration management (CM) is a service that runs and manages configuration management servers.
69  *
70  * </p
71  *
72  * <b>Glossary of terms</b>
73  *
74  * </p <ul> <li>
75  *
76  * <b>Server</b>: A configuration management server that can be highly-available. The configuration management server runs
77  * on an Amazon Elastic Compute Cloud (EC2) instance, and may use various other AWS services, such as Amazon Relational
78  * Database Service (RDS) and Elastic Load Balancing. A server is a generic abstraction over the configuration manager that
79  * you want to use, much like Amazon RDS. In AWS OpsWorks CM, you do not start or stop servers. After you create servers,
80  * they continue to run until they are
81  *
82  * deleted> </li> <li>
83  *
84  * <b>Engine</b>: The engine is the specific configuration manager that you want to use. Valid values in this release
85  * include <code>Chef</code> and
86  *
87  * <code>Puppet</code>> </li> <li>
88  *
89  * <b>Backup</b>: This is an application-level backup of the data that the configuration manager stores. AWS OpsWorks CM
90  * creates an S3 bucket for backups when you launch the first server. A backup maintains a snapshot of a server's
91  * configuration-related attributes at the time the backup
92  *
93  * starts> </li> <li>
94  *
95  * <b>Events</b>: Events are always related to a server. Events are written during server creation, when health checks run,
96  * when backups are created, when system maintenance is performed, etc. When you delete a server, the server's events are
97  * also
98  *
99  * deleted> </li> <li>
100  *
101  * <b>Account attributes</b>: Every account has attributes that are assigned in the AWS OpsWorks CM database. These
102  * attributes store information about configuration limits (servers, backups, etc.) and your customer account.
103  *
104  * </p </li> </ul>
105  *
106  * <b>Endpoints</b>
107  *
108  * </p
109  *
110  * AWS OpsWorks CM supports the following endpoints, all HTTPS. You must connect to one of the following endpoints. Your
111  * servers can only be accessed or managed within the endpoint in which they are
112  *
113  * created> <ul> <li>
114  *
115  * opsworks-cm.us-east-1.amazonaws.co> </li> <li>
116  *
117  * opsworks-cm.us-west-2.amazonaws.co> </li> <li>
118  *
119  * opsworks-cm.eu-west-1.amazonaws.co> </li> </ul>
120  *
121  * <b>Throttling limits</b>
122  *
123  * </p
124  *
125  * All API operations allow for five requests per second with a burst of 10 requests per
