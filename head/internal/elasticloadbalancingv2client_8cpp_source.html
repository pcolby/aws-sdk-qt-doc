99 /**
100  * @class ElasticLoadBalancingv2Client
101  *
102  * @brief Client for Elastic Load Balancing (Elastic Load Balancing v2)
103  *
104  * <fullname>Elastic Load Balancing</fullname>
105  *
106  * A load balancer distributes incoming traffic across targets, such as your EC2 instances. This enables you to increase
107  * the availability of your application. The load balancer also monitors the health of its registered targets and ensures
108  * that it routes traffic only to healthy targets. You configure your load balancer to accept incoming traffic by
109  * specifying one or more listeners, which are configured with a protocol and port number for connections from clients to
110  * the load balancer. You configure a target group with a protocol and port number for connections from the load balancer
111  * to the targets, and with health check settings to be used when checking the health status of the
112  *
113  * targets>
114  *
115  * Elastic Load Balancing supports the following types of load balancers: Application Load Balancers, Network Load
116  * Balancers, and Classic Load
117  *
118  * Balancers>
119  *
120  * An Application Load Balancer makes routing and load balancing decisions at the application layer (HTTP/HTTPS). A Network
121  * Load Balancer makes routing and load balancing decisions at the transport layer (TCP). Both Application Load Balancers
122  * and Network Load Balancers can route requests to one or more ports on each EC2 instance or container instance in your
123  * virtual private cloud
124  *
125  * (VPC)>
126  *
127  * A Classic Load Balancer makes routing and load balancing decisions either at the transport layer (TCP/SSL) or the
128  * application layer (HTTP/HTTPS), and supports either EC2-Classic or a VPC. For more information, see the <a
129  * href="http://docs.aws.amazon.com/elasticloadbalancing/latest/userguide/">Elastic Load Balancing User
130  *
131  * Guide</a>>
132  *
133  * This reference covers the 2015-12-01 API, which supports Application Load Balancers and Network Load Balancers. The
134  * 2012-06-01 API supports Classic Load
135  *
136  * Balancers>
137  *
138  * To get started, complete the following
139  *
140  * tasks> <ol> <li>
141  *
142  * Create a load balancer using
143  *
144  * <a>CreateLoadBalancer</a>> </li> <li>
145  *
146  * Create a target group using
147  *
148  * <a>CreateTargetGroup</a>> </li> <li>
149  *
150  * Register targets for the target group using
151  *
152  * <a>RegisterTargets</a>> </li> <li>
153  *
154  * Create one or more listeners for your load balancer using
155  *
156  * <a>CreateListener</a>> </li> </ol>
157  *
158  * To delete a load balancer and its related resources, complete the following
159  *
160  * tasks> <ol> <li>
161  *
162  * Delete the load balancer using
163  *
164  * <a>DeleteLoadBalancer</a>> </li> <li>
165  *
166  * Delete the target group using
167  *
168  * <a>DeleteTargetGroup</a>> </li> </ol>
169  *
