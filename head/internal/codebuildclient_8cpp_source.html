64  * @class CodeBuildClient
65  *
66  * @brief Client for AWS CodeBuild
67  *
68  * <fullname>AWS CodeBuild</fullname>
69  *
70  * AWS CodeBuild is a fully managed build service in the cloud. AWS CodeBuild compiles your source code, runs unit tests,
71  * and produces artifacts that are ready to deploy. AWS CodeBuild eliminates the need to provision, manage, and scale your
72  * own build servers. It provides prepackaged build environments for the most popular programming languages and build
73  * tools, such as Apache Maven, Gradle, and more. You can also fully customize build environments in AWS CodeBuild to use
74  * your own build tools. AWS CodeBuild scales automatically to meet peak build requests, and you pay only for the build
75  * time you consume. For more information about AWS CodeBuild, see the <i>AWS CodeBuild User
76  *
77  * Guide</i>>
78  *
79  * AWS CodeBuild supports these
80  *
81  * operations> <ul> <li>
82  *
83  * <code>BatchDeleteBuilds</code>: Deletes one or more
84  *
85  * builds> </li> <li>
86  *
87  * <code>BatchGetProjects</code>: Gets information about one or more build projects. A <i>build project</i> defines how AWS
88  * CodeBuild will run a build. This includes information such as where to get the source code to build, the build
89  * environment to use, the build commands to run, and where to store the build output. A <i>build environment</i>
90  * represents a combination of operating system, programming language runtime, and tools that AWS CodeBuild will use to run
91  * a build. Also, you can add tags to build projects to help manage your resources and
92  *
93  * costs> </li> <li>
94  *
95  * <code>CreateProject</code>: Creates a build
96  *
97  * project> </li> <li>
98  *
99  * <code>CreateWebhook</code>: For an existing AWS CodeBuild build project that has its source code stored in a GitHub
100  * repository, enables AWS CodeBuild to begin automatically rebuilding the source code every time a code change is pushed
101  * to the
102  *
103  * repository> </li> <li>
104  *
105  * <code>UpdateWebhook</code>: Changes the settings of an existing
106  *
107  * webhook> </li> <li>
108  *
109  * <code>DeleteProject</code>: Deletes a build
110  *
111  * project> </li> <li>
112  *
113  * <code>DeleteWebhook</code>: For an existing AWS CodeBuild build project that has its source code stored in a GitHub
114  * repository, stops AWS CodeBuild from automatically rebuilding the source code every time a code change is pushed to the
115  *
116  * repository> </li> <li>
117  *
118  * <code>ListProjects</code>: Gets a list of build project names, with each build project name representing a single build
119  *
120  * project> </li> <li>
121  *
122  * <code>UpdateProject</code>: Changes the settings of an existing build
123  *
124  * project> </li> <li>
125  *
126  * <code>BatchGetBuilds</code>: Gets information about one or more
127  *
128  * builds> </li> <li>
129  *
130  * <code>ListBuilds</code>: Gets a list of build IDs, with each build ID representing a single
131  *
132  * build> </li> <li>
133  *
134  * <code>ListBuildsForProject</code>: Gets a list of build IDs for the specified build project, with each build ID
135  * representing a single
136  *
137  * build> </li> <li>
138  *
139  * <code>StartBuild</code>: Starts running a
140  *
141  * build> </li> <li>
142  *
143  * <code>StopBuild</code>: Attempts to stop running a
144  *
145  * build> </li> <li>
