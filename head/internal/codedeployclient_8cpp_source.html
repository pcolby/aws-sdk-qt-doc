111 /**
112  * @class CodeDeployClient
113  *
114  * @brief Client for AWS CodeDeploy
115  *
116  * <fullname>AWS CodeDeploy</fullname>
117  *
118  * AWS CodeDeploy is a deployment service that automates application deployments to Amazon EC2 instances, on-premises
119  * instances running in your own facility, or serverless AWS Lambda
120  *
121  * functions>
122  *
123  * You can deploy a nearly unlimited variety of application content, such as an updated Lambda function, code, web and
124  * configuration files, executables, packages, scripts, multimedia files, and so on. AWS CodeDeploy can deploy application
125  * content stored in Amazon S3 buckets, GitHub repositories, or Bitbucket repositories. You do not need to make changes to
126  * your existing code before you can use AWS
127  *
128  * CodeDeploy>
129  *
130  * AWS CodeDeploy makes it easier for you to rapidly release new features, helps you avoid downtime during application
131  * deployment, and handles the complexity of updating your applications, without many of the risks associated with
132  * error-prone manual
133  *
134  * deployments>
135  *
136  * <b>AWS CodeDeploy Components</b>
137  *
138  * </p
139  *
140  * Use the information in this guide to help you work with the following AWS CodeDeploy
141  *
142  * components> <ul> <li>
143  *
144  * <b>Application</b>: A name that uniquely identifies the application you want to deploy. AWS CodeDeploy uses this name,
145  * which functions as a container, to ensure the correct combination of revision, deployment configuration, and deployment
146  * group are referenced during a
147  *
148  * deployment> </li> <li>
149  *
150  * <b>Deployment group</b>: A set of individual instances or CodeDeploy Lambda applications. A Lambda deployment group
151  * contains a group of applications. An EC2/On-premises deployment group contains individually tagged instances, Amazon EC2
152  * instances in Auto Scaling groups, or both.
153  *
154  * </p </li> <li>
155  *
156  * <b>Deployment configuration</b>: A set of deployment rules and deployment success and failure conditions used by AWS
157  * CodeDeploy during a
158  *
159  * deployment> </li> <li>
160  *
161  * <b>Deployment</b>: The process and the components used in the process of updating a Lambda function or of installing
162  * content on one or more instances.
163  *
164  * </p </li> <li>
165  *
166  * <b>Application revisions</b>: For an AWS Lambda deployment, this is an AppSpec file that specifies the Lambda function
167  * to update and one or more functions to validate deployment lifecycle events. For an EC2/On-premises deployment, this is
168  * an archive file containing source content—source code, web pages, executable files, and deployment scripts—along with an
169  * AppSpec file. Revisions are stored in Amazon S3 buckets or GitHub repositories. For Amazon S3, a revision is uniquely
170  * identified by its Amazon S3 object key and its ETag, version, or both. For GitHub, a revision is uniquely identified by
171  * its commit
172  *
173  * ID> </li> </ul>
174  *
175  * This guide also contains information to help you get details about the instances in your deployments, to make
176  * on-premises instances available for AWS CodeDeploy deployments, and to get details about a Lambda function
177  *
178  * deployment>
179  *
180  * <b>AWS CodeDeploy Information Resources</b>
181  *
182  * </p <ul> <li>
183  *
184  * <a href="http://docs.aws.amazon.com/codedeploy/latest/userguide">AWS CodeDeploy User Guide</a>
185  *
186  * </p </li> <li>
