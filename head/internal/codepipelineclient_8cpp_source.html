85 /**
86  * @class CodePipelineClient
87  *
88  * @brief Client for AWS CodePipeline
89  *
90  * <fullname>AWS CodePipeline</fullname>
91  *
92  * <b>Overview</b>
93  *
94  * </p
95  *
96  * This is the AWS CodePipeline API Reference. This guide provides descriptions of the actions and data types for AWS
97  * CodePipeline. Some functionality for your pipeline is only configurable through the API. For additional information, see
98  * the <a href="http://docs.aws.amazon.com/codepipeline/latest/userguide/welcome.html">AWS CodePipeline User
99  *
100  * Guide</a>>
101  *
102  * You can use the AWS CodePipeline API to work with pipelines, stages, actions, gates, and transitions, as described
103  *
104  * below>
105  *
106  * <i>Pipelines</i> are models of automated release processes. Each pipeline is uniquely named, and consists of actions,
107  * gates, and stages.
108  *
109  * </p
110  *
111  * You can work with pipelines by
112  *
113  * calling> <ul> <li>
114  *
115  * <a>CreatePipeline</a>, which creates a uniquely-named
116  *
117  * pipeline> </li> <li>
118  *
119  * <a>DeletePipeline</a>, which deletes the specified
120  *
121  * pipeline> </li> <li>
122  *
123  * <a>GetPipeline</a>, which returns information about the pipeline structure and pipeline metadata, including the pipeline
124  * Amazon Resource Name
125  *
126  * (ARN)> </li> <li>
127  *
128  * <a>GetPipelineExecution</a>, which returns information about a specific execution of a
129  *
130  * pipeline> </li> <li>
131  *
132  * <a>GetPipelineState</a>, which returns information about the current state of the stages and actions of a
133  *
134  * pipeline> </li> <li>
135  *
136  * <a>ListPipelines</a>, which gets a summary of all of the pipelines associated with your
137  *
138  * account> </li> <li>
139  *
140  * <a>ListPipelineExecutions</a>, which gets a summary of the most recent executions for a
141  *
142  * pipeline> </li> <li>
143  *
144  * <a>StartPipelineExecution</a>, which runs the the most recent revision of an artifact through the
145  *
146  * pipeline> </li> <li>
147  *
148  * <a>UpdatePipeline</a>, which updates a pipeline with edits or changes to the structure of the
149  *
150  * pipeline> </li> </ul>
151  *
152  * Pipelines include <i>stages</i>, which are logical groupings of gates and actions. Each stage contains one or more
153  * actions that must complete before the next stage begins. A stage will result in success or failure. If a stage fails,
154  * then the pipeline stops at that stage and will remain stopped until either a new version of an artifact appears in the
155  * source location, or a user takes action to re-run the most recent artifact through the pipeline. You can call
156  * <a>GetPipelineState</a>, which displays the status of a pipeline, including the status of stages in the pipeline, or
157  * <a>GetPipeline</a>, which returns the entire structure of the pipeline, including the stages of that pipeline. For more
158  * information about the structure of stages and actions, also refer to the <a
159  * href="http://docs.aws.amazon.com/codepipeline/latest/userguide/pipeline-structure.html">AWS CodePipeline Pipeline
160  * Structure
161  *
162  * Reference</a>>
163  *
164  * Pipeline stages include <i>actions</i>, which are categorized into categories such as source or build actions performed
165  * within a stage of a pipeline. For example, you can use a source action to import artifacts into a pipeline from a source
166  * such as Amazon S3. Like stages, you do not work with actions directly in most cases, but you do define and interact with
167  * actions when working with pipeline operations such as <a>CreatePipeline</a> and <a>GetPipelineState</a>.
168  *
169  * </p
170  *
171  * Pipelines also include <i>transitions</i>, which allow the transition of artifacts from one stage to the next in a
172  * pipeline after the actions in one stage
173  *
174  * complete>
175  *
176  * You can work with transitions by
177  *
178  * calling> <ul> <li>
179  *
180  * <a>DisableStageTransition</a>, which prevents artifacts from transitioning to the next stage in a
181  *
182  * pipeline> </li> <li>
183  *
184  * <a>EnableStageTransition</a>, which enables transition of artifacts between stages in a pipeline.
185  *
186  * </p </li> </ul>
187  *
188  * <b>Using the API to integrate with AWS CodePipeline</b>
189  *
190  * </p
191  *
192  * For third-party integrators or developers who want to create their own integrations with AWS CodePipeline, the expected
193  * sequence varies from the standard API user. In order to integrate with AWS CodePipeline, developers will need to work
194  * with the following
195  *
196  * items>
197  *
198  * <b>Jobs</b>, which are instances of an action. For example, a job for a source action might import a revision of an
199  * artifact from a source.
200  *
201  * </p
202  *
203  * You can work with jobs by
204  *
205  * calling> <ul> <li>
206  *
207  * <a>AcknowledgeJob</a>, which confirms whether a job worker has received the specified
208  *
209  * job> </li> <li>
210  *
211  * <a>GetJobDetails</a>, which returns the details of a
212  *
213  * job> </li> <li>
214  *
215  * <a>PollForJobs</a>, which determines whether there are any jobs to act upon,
216  *
217  * </p </li> <li>
218  *
219  * <a>PutJobFailureResult</a>, which provides details of a job failure,
220  *
221  * an> </li> <li>
222  *
223  * <a>PutJobSuccessResult</a>, which provides details of a job
224  *
225  * success> </li> </ul>
226  *
227  * <b>Third party jobs</b>, which are instances of an action created by a partner action and integrated into AWS
228  * CodePipeline. Partner actions are created by members of the AWS Partner
229  *
230  * Network>
231  *
232  * You can work with third party jobs by
233  *
234  * calling> <ul> <li>
235  *
236  * <a>AcknowledgeThirdPartyJob</a>, which confirms whether a job worker has received the specified
237  *
238  * job> </li> <li>
239  *
240  * <a>GetThirdPartyJobDetails</a>, which requests the details of a job for a partner
241  *
242  * action> </li> <li>
243  *
244  * <a>PollForThirdPartyJobs</a>, which determines whether there are any jobs to act upon,
245  *
246  * </p </li> <li>
247  *
248  * <a>PutThirdPartyJobFailureResult</a>, which provides details of a job failure,
249  *
250  * an> </li> <li>
251  *
252  * <a>PutThirdPartyJobSuccessResult</a>, which provides details of a job
253  */
