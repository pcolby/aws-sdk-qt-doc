89 /**
90  * @class ElasticLoadBalancingClient
91  *
92  * @brief Client for Elastic Load Balancing
93  *
94  * <fullname>Elastic Load Balancing</fullname>
95  *
96  * A load balancer can distribute incoming traffic across your EC2 instances. This enables you to increase the availability
97  * of your application. The load balancer also monitors the health of its registered instances and ensures that it routes
98  * traffic only to healthy instances. You configure your load balancer to accept incoming traffic by specifying one or more
99  * listeners, which are configured with a protocol and port number for connections from clients to the load balancer and a
100  * protocol and port number for connections from the load balancer to the
101  *
102  * instances>
103  *
104  * Elastic Load Balancing supports three types of load balancers: Application Load Balancers, Network Load Balancers, and
105  * Classic Load Balancers. You can select a load balancer based on your application needs. For more information, see the <a
106  * href="http://docs.aws.amazon.com/elasticloadbalancing/latest/userguide/">Elastic Load Balancing User
107  *
108  * Guide</a>>
109  *
110  * This reference covers the 2012-06-01 API, which supports Classic Load Balancers. The 2015-12-01 API supports Application
111  * Load Balancers and Network Load
112  *
113  * Balancers>
114  *
115  * To get started, create a load balancer with one or more listeners using <a>CreateLoadBalancer</a>. Register your
116  * instances with the load balancer using
117  *
118  * <a>RegisterInstancesWithLoadBalancer</a>>
119  *
120  * All Elastic Load Balancing operations are <i>idempotent</i>, which means that they complete at most one time. If you
121  * repeat an operation, it succeeds with a 200 OK response
122  */
