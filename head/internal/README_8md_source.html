11 The AWS SDK for Qt provides a modern [Qt] (version 5.1 or later) interface for [AWS].
12 
13 The library is still in early stages of development, but already includes much
14 of the basic framework plumbing required for implementing asynchronous AWS
15 clients, including generic classes for managing the building of AWS requests
16 (including all five AWS signature types) and processing of AWS responses.
17 
18 The first [0.1.0](https://github.com/pcolby/aws-sdk-qt/releases/tag/v0.1.0)
19 release included just the signature, endpoint and region classes. But that
20 release was [100% documented](http://pcolby.github.io/aws-sdk-qt/0.1.0/api/annotated.html)
21 (including the [internal code](http://pcolby.github.io/aws-sdk-qt/0.1.0/internal/annotated.html))
22 and 100% [unit tested](http://pcolby.github.io/aws-sdk-qt/0.1.0/coverage/).
23 
24 Work then progressed for the SQS client classes - the basics of which are in
25 place now (see the [0.2.0 pre-release](https://github.com/pcolby/aws-sdk-qt/releases/tag/v0.2.0)),
26 however that work was put on hold when Amazon released their official
27 [AWS SDK for C++](https://github.com/aws/aws-sdk-cpp "aws-sdk-cpp"), while this
28 project re-evaluated its value proposition, as well as a potential shift to
29 becoming a light Qt-based wrapper for Amazon's SDK (which was not deemed a good
30 fit in the end).
31 
32 However, now that Amazon has made public the API descriptions used to
33 generate most of the C++ SDK, effort is now being put into a Qt-based code
34 generator that will create the bulk of the service client and model classes from
35 those API descriptions.
36 
37 Note that the API descriptions do not cover things like the signature, endpoint
38 and region classes, which is where the bulk of the development effort has been
39 to date, making all the investment in this project so far still completely
40 worthwhile :)

42 ## Requirements
43 ### Qt 5.1+
44 AWS SDK for Qt requires Qt 5.1 or later, as the library makes use of Qt's
45 QMessageAuthenticationCode class for generating HMAC codes for AWS request
46 signatures, and that class was first added in Qt 5.1.0.
47 
48 Backporting QMessageAuthenticationCode to Qt 5.0 is [relatively simple](
49 https://github.com/pcolby/aws-sdk-qt/blob/196b5cd5e571d6883d288705234770a5e606d1f0/.travis.yml#L34),
50 but outside the scope of this project.
51 
52 Note, the [code-generation](code-generation) sub-project requires Qt
53 5.10+ and [Grantlee](https://github.com/steveire/grantlee), but this is
54 for development of SDK itself, not for devlopers that use the SDK.

56 ## Building
57 This library supports out-of-source builds (highly recommended), for example:
58 
59 ~~~{.sh}
60 mkdir -p <tmp-build-dir>
61 pushd <tmp-build-dir>
62 qmake -Wall -Wlogic -Wparser CONFIG+=debug_and_release <path-to-cloned-repo>
63 make # or `nmake` on Windows
64 make check
65 popd
66 ~~~
